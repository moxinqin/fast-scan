📖 工具概述
本工具是一款高效、灵活的fast-scan漏洞扫描工具，核心优势在于灵活性高、扩展性强，且具备严格的包发送校验机制，可确保不会发送重复请求包超过三次，有效避免对目标服务器造成不必要的压力，同时提升扫描效率与访问网站失败次数过多时自动终止相关扫描任务。
工具支持单个URL扫描与批量URL扫描两种模式，可自定义扫描范围、漏洞级别、并发数等参数，适配不同场景下的扫描需求，同时支持代理配置、结果导出，满足多样化使用场景。

🔧 命令行参数说明
工具采用命令行交互模式，参数分为「必选互斥参数」和「可选参数」，可根据实际需求灵活配置，具体说明如下：
3.1 必选互斥参数（二选一，必须传入一个）
-u：指定单个URL地址，用于对单个目标进行漏洞扫描
  示例：-u https://example.com
  说明：该示例URL为文档演示专用，请勿用于实际操作扫描。
-list：指定URL文件路径（文件内需一行一个URL），用于批量扫描多个目标
  示例：-list url.txt
注意：-u 和 -list 不可同时使用，也不可都不传入，否则会报错。
3.2 可选参数（非必传，有默认值，可按需配置）
-o：指定扫描结果输出文件路径，扫描完成后结果会写入该文件
  示例：-o result.txt
-proxy：指定代理地址，用于通过代理进行扫描（默认无代理）
  示例：-proxy http://127.0.0.1:8080
  说明：若使用该示例代理地址报错「URL拼写可能存在错误，请检查」，请核实代理地址的正确性、端口是否开放。
-c：指定同时处理的URL数量，int类型，默认值为25，可根据电脑性能调整
 示例：-c 50
-threads：指定每个URL的并发线程数，int类型，默认值为10，用于控制单个URL的扫描速度
  示例：-threads 15
-scope：指定扫描范围，多个范围用逗号隔开，默认为全部（需填写main-call.py的相对路径）
  示例：-scope scan1,scan2
-level：指定需要扫描的漏洞级别，多个级别用逗号隔开，默认扫描全部级别（极危,严重,高危,中危,低危,信息）
  示例：-level 高危,中危

✨ 工具核心特点
1. 灵活性高：支持单个/批量URL扫描，可自定义同时处理的URL数量、单个URL的并发线程数量、漏洞级别、扫描范围等参数，适配不同扫描场景（如快速扫描、深度扫描）；支持代理配置，可应对不同网络环境。
2. 扩展性强：可根据需求扩展扫描功能（如数据库漏洞扫描、LDAP漏洞扫描）；代码结构清晰，便于后续新增扫描模块、扩展漏洞检测规则。
3. 去重超强：内置严格的请求校验机制，可避免对同一URL、同一漏洞发送大量不必要的重复请求包，既减少无效扫描操作，又降低对目标服务器的负载，提升扫描效率（无论是GET、POST、PUT、DELETE等等，在响应内容一样的基础上重复发包最多三次）。

📝 使用示例
# 示例1：单个URL扫描，指定输出文件和高危级别
python -u main-call.py -u https://example.com -o scan_result.txt -level 高危
# 示例2：批量URL扫描，指定代理、同时处理URL数和扫描范围
python -u main-call.py -list url.txt -proxy http://127.0.0.1:8080 -c 30 -scope web
